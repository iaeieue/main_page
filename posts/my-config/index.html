遵循最低权限原则配置网站目录#

获取 nginx worker process 进程用户

ps aux | grep nginx

将网站目录所有者和组改为与 nginx worker process 相同

chown -R www-data:www-data /var/www/example.com

将网站目录权限递归设置为 755，文件权限设置为 644

find /var/www/example.com -type d -exec chmod 755 {} \;
find /var/www/example.com -type f -exec chmod 644 {} \;

额外配置#

配置 https 强制跳转并禁止纯 IP 访问（统一返回 444 响应）

/etc/nginx/sites-available/example.com
http {
    server {
        listen 80 default_server;
        listen [::]:80 default_server;
        server_name _;

        return 444;
        error_page 400 =444 /;
        location = / {
            return 444;
        }
    }

    server {
        listen 80;
        listen [::]:80;
        server_name *.example.com;

        return 301 https://$host$request_uri;
    }

    server {
        listen 443 ssl default_server;
        listen [::]:443 ssl default_server;
        server_name _;
        http2 on;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_reject_handshake on;

        error_page 400 497 =444 /;
        location = / {
            return 444;
        }
    }
}

获取 nginx 版本信息

nginx -v

根据输出的版本信息在 https://nginx.org/en/download.html 下载对应版本的包体

编译并动态加载 ngx_brotli 模块#

安装编译库

apt install build-essential libpcre3 libpcre3-dev zlib1g \
    zlib1g-dev cmake

下载并解压对应版本的 nginx 包体

wget -P $HOME https://nginx.org/download/nginx-1.26.2.tar.gz
tar -xzvf nginx-1.26.2.tar.gz

根据 官方文档 说明进行动态加载编译，且需要进行依赖项的构建

构建完成后进行动态加载

cd nginx-1.26.2
./configure --with-compat --add-dynamic-module=$HOME/ngx_brotli
make modules

cp objs/*.so /etc/nginx/modules
nano /etc/nginx/modules-available/50-mod-http_brotli.conf
ln -s /etc/nginx/modules-available/50-mod-http_brotli.conf \
    /etc/nginx/modules-enabled
/etc/nginx/modules-available/50-mod-http_brotli.conf
load_module /etc/nginx/modules/ngx_http_brotli_filter_module.so;
load_module /etc/nginx/modules/ngx_http_brotli_static_module.so;

cake QoS#

测试认为仅针对 sysctl 进行配置似乎无法正确应用到网口上，需要额外针对网口进行参数优化

/usr/local/bin/cake-qos.sh
#!/bin/sh

tc qdisc del dev eth0 root 2>/dev/null
tc qdisc add dev eth0 root cake \
    bandwidth 530Mbit \
    diffserv4 \
    triple-isolate \
    nonat \
    wash \
    ack-filter \
    split-gso \
    rtt 40ms \
    raw

配置持久化应用

/etc/systemd/system/cake-qos.service
[Unit]
Description=Apply CAKE QoS configuration
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/cake-qos.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
  • systemctl daemon-reload
  • systemctl enable cake-qos
  • systemctl start cake-qos
